2. FONT VARIABLE RESOLUTION FAILURE
   Severity: CRITICAL
   Impact: All 20 pages (40 instances total)
   
   Issue: CSS uses variables that are not defined or not resolving:
     - var(--font-display) [should = Cormorant Garamond]
     - var(--font-body) [should = DM Sans]

   Expected: Direct font-family declarations or globally defined CSS variables
   
   Action Required: 
     a) Define global CSS variables in a shared stylesheet or :root
     b) Ensure variables are available on ALL pages
     c) Verify fallback font stacks are in place
   
   Affected Pages: ALL 20

3. HEADING FONT WEIGHTS INCORRECT
   Severity: CRITICAL (Typography)
   Impact: ~30+ instances across multiple pages
   
   Issue: h1 and h2 elements using insufficient font-weight values:
     - Found: font-weight 400, 500, 600
     - Expected: font-weight 700 or bold
   
   Examples:
     - 404.html: h1 = 600, h2 = undefined
     - about.html: h1 = 400, h2 = 500 or 400
     - adv.html: h1 = 600, h2 = 600
   
   Impact: Headings appear too light, reducing visual hierarchy and readability
   
   Action Required: Update all h1 and h2 to font-weight: 700
   
   Affected Pages: 404.html, about.html, adv.html, blog.html, contact.html,
                  disclosures.html, forgot-password.html, get-started.html,
                  index.html, login.html, planning-*.html, privacy.html,
                  terms.html, thank-you.html, veterans.html

HIGH PRIORITY ISSUES (Accessibility Concerns)
================================================================================

4. BODY TEXT SIZE BELOW MINIMUM
   Severity: HIGH (WCAG)
   Impact: ~50 instances across pages
   
   Issue: Some body/paragraph text sizes fall below 0.95rem (14px) minimum
   
   Found Sizes:
     - 0.76rem (12.2px)
     - 0.78rem (12.5px)
     - 0.8rem (12.8px)
     - 0.82rem (13.1px)
     - 0.84rem (13.4px)
     - 0.86rem (13.8px)
     - 0.88rem (14.1px) [borderline]
     - 0.9rem (14.4px) [borderline]
     - 12px-15px (various)
   
   WCAG 2.1 Requirement: Minimum 0.95rem (14px) for normal text
   Accessibility Impact: Reduced readability, especially for users 50+ and those
                        with vision impairments
   
   Affected Pages: about.html, autism-resource-navigator.html, blog.html,
                  contact.html, and others
   
   Action Required: Audit all <p>, <body>, and text elements. Update any below
                   0.95rem to meet accessibility minimum.

5. LOW CONTRAST RGBA VALUES
   Severity: HIGH (WCAG AAA concern, some AA failures)
   Impact: 887 instances across all pages
   
   Issue: Widespread use of very light rgba() colors with low alpha values
   
   Alpha Values Below 0.4 (highest risk):
     - Alpha 0.02-0.08: 330 instances (extremely low - avoid)
     - Alpha 0.1-0.2: 160 instances (very low - at risk)
     - Alpha 0.25-0.3: 164 instances (low - test on actual backgrounds)
     - Alpha 0.35-0.38: 38 instances (borderline)
   
   WCAG 2.1 Level AA Requirement: 4.5:1 contrast ratio for normal text
   
   Examples of problematic values:
     - rgba(33, 160, 219, 0.08) - 8% opacity blue on light = very faint
     - rgba(255, 255, 255, 0.05) - 5% opacity white on dark = nearly invisible
     - rgba(0, 0, 0, 0.3) - 30% opacity black on light = may fail AA
   
   Recommendation: 
     a) Audit each rgba color against its background
     b) Increase alpha to at least 0.4 for any text or essential UI elements
     c) For decorative overlays, current values may be acceptable
   
   Affected Pages: ALL 20 (design-wide issue)
